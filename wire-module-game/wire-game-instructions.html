Wire Module Game (temporary title)

Any questions, comments, or suggestions can be sent to nkrim62@gmail.com

Instructions

    Click and drag to draw wires between grid cells, drag modules from the toolbar onto the board (or a wire module to delete any other module on a cell), and right-click a cell to gain access to extra options (like signal colors, diode output directions, and toggle and rotator augmentations).


Base Mechanics

    Create a mechanism using modules connected via wires on the grid board. Send and recieve signals to communicate and create responsive systems that can perform more compicated logic and sequences. In the current build, all levels are pattern levels in which you have to send certain combinations of signals together in one single tick (I plan on making more types of levels but I'm kind of stuck on deciding what would be good, so any ideas are welcome n_n).
     Each run of a mechanism will start with a power tick where all power modules send power out to all of their connected wires, and if any signals are sent (explained below), a series of resolution ticks will occur until all signals are resolved, after which another power tick will occur, ad infinitum.


Signals

    Signals are sent when an emittor module recieves power. Each emittor module has a channel (selected from the right-click menu), and when a module is powered it will send one signal for each powered input on the module during that tick, for every emittor module on the board.
    Reciever modules and toggle and rotator augmentations recieve signals from a signal manager, which queues up the signals for each colored channel and sends them out one tick at a time as resolution ticks (so if a blue and a green signal are sent, then one resolution tick will proceed afterwards which will resolve blue and green recievers and augmentations, however if two green signals are sent, two resolution ticks will proceed each one sending just one blue signal to recievers and augmentations). If there are no signals left in the signal manager's queue, then a power tick will follow. It should be noted that there will never be an instance where a reciever module and a power module fire at the same time.
    For pattern levels, it considers consecutive signal emissions in a tick as one line of the pattern sequence, and will evaluate the next line on the next tick with signal emissions (any number of ticks can occur between these).


Modules

  • Wire Module

    Base module, will freely allow flow of power through all connected wires.

  • Power Module

    Will send power to all connected wires once per power tick.

  • OR Diode

    When power is sent to any non-output side of the module, power is the sent to all output wires (output wires determined from right-click menu).

  • AND Diode

    When power is sent to all non-output sides of the module that have incoming wires, power is the sent to all output wires (output wires determined from right-click menu).

  • Emittor Module

    Will send one signal on the modules channel color (changeable from the right-click menu) for each connected wire that is powered.

  • Reciever Module

    When a signal of the same channel color (changeable from the right-click menu) is sent from the signal manager's queue, this module will emit power in the same as a power module but instead during a resolution tick.

  • Bridge Module

    Acts like a wire module, but each of its two axes are separated from eachother (so if all four wires are connected, and power comes in from the left, then power will only be sent to the right, and vice versa and equivalently for the up and down directions).


Augmentations (can be added to any grid cell from the right-click menu)

Note: All augmentation actions are evaluated before reciever actions

  • Toggle Augmentation

    When a signal is sent of the same channel color as this augmentation, it will toggle the entire cell between an on and off state. When the cell is off, no power will be sent to and from wires in the cell, but augmentations will still evaluate.

  • Rotator Augmentation

    When a signal is sent of the same channel color as this augmentation, it will rotate the entire cell (wires and modules) 90deg clockwise.


Level Types

  • Pattern Levels

    Pattern levels request a repeated sequence of signal emission combinations per tick (ticks with no signal emissions are ignored). Any extraneous signal emissions (of either the same channel or other unique channels) will break the pattern. The ordering of the colors in a pattern line are arbitrary for solving the level.


General tips that I've found helpful

    For pattern levels, the main tip I have is to either implement the entire level using power modules and augmentatoins (so that power is only sent on power ticks), OR to have one power module that is hooked up to the initial emittors for the first line of a pattern, and then resolve all further lines of the pattern using an infinite loop of reciever modules, so the entire run will have one power tick at the start and infinite signal resolution ticks following that. Power modules will never fire on the same tick as reciever modules, so while you can coordinate intermittently with resolution ticks, then have the signal manager flush itself so that another power tick can occur, this will not work out easily for many of the levels included (though it will work out for some).
